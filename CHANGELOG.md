## [0.1.1] - 2026-06-20

### Added
- Додано централізоване логування застосунку (console + file logging).
- Додано middleware для логування всіх HTTP запитів (метод, шлях, статус, час виконання, IP).
- Додано `lifespan` для управління життєвим циклом FastAPI застосунку.
- Додано бібліотеку `pydantic-settings`
- Додано файл .env.example з прикладом конфігурації застосунку.
- Додано підтримку конфігурації через файл .env.
- Додано централізовану конфігурацію застосунку через settings.py.
- Додано налаштування параметрів бази даних через Pydantic Settings.
- Додано конфігурацію `Dockerfile` для контейнеризації FastAPI застосунку на базі легкого образу Python Slim.
- Додано файл `docker-compose.yml` для мультиконтейнерного розгортання застосунку та бази даних PostgreSQL 16.
- Додано автоматичне монтування локальної директорії (`volumes`) та режим `--reload` для зручної локальної розробки в Docker.
- Інтегровано та повністю налаштовано Alembic для асинхронних міграцій через `asyncpg`.
- Згенеровано першу базову міграцію для ініціалізації структури бази даних.
- Додано автоматичну перевірку з'єднання з базою даних (healthcheck) при старті через `check_db_connection`.
- Створено базовий шаблон `base.html` на Jinja2 для уникнення дублювання HTML коду.

### Changed
- Оновлено ініціалізацію логера для підтримки uvicorn логів.
- Додано в `lifespan` логування базової директорії та url бази даних
- Оновлено команду запуску Uvicorn в Docker-контейнері для роботи з кореневим файлом `main:app`.

### Fixed
- Покращено стабільність логування при reload режимі uvicorn.

## [0.2.0] - 2026-06-20

### Added
- Додано властивості `created_at_local` та `updated_at_local` для автоматичної конвертації часу сервера з UTC у Київський часовий пояс при відображенні.
- Додано SQLAlchemy модель `User` з полями для email, хешованого пароля, дати народження та статусу активності.
- Згенеровано та застосовано асинхронну міграцію Alembic для створення таблиці `users` у базі даних PostgreSQL.
- Додано `SECRET_KEY_JWT` в `.env` файл та `settings`
- Додано router `/auth`
- Додано `messages` в `config`
- Додано модель Token для подальшого використання
- Додано шлях `/auth/register` для реєстрації нового користувача
- Додано повноцінну сторінку реєстрації користувачів (`GET /auth/register`).
- Додано технічний ендпоінт для обробки даних реєстрації (`POST /api/auth/register`).
- Створено HTML-шаблон `register.html` у мінімалістичному стилі з формою створення акаунта.
- Впроваджено JavaScript (`fetch` API) у шаблон реєстрації для асинхронної відправки JSON даних та динамічного виведення помилок валідації.
- Додано кнопку "Реєстрація" на головну сторінку `index.html` поруч із кнопкою входу.
- Додано повноцінну сторінку авторизації користувачів (`GET /auth/login`).
- Додано технічний ендпоінт для перевірки пароля та входу (`POST /api/auth/login`) з використанням `OAuth2PasswordRequestForm`.
- Додано роут для виходу із системи (`GET /auth/logout`), який анулює сесію та видаляє куки доступу.
- Створено HTML-шаблон сторінки входу `login.html` у мінімалістичному стилі з обробкою AJAX-форм.
- Додано модель `Token` та шар репозиторію `UserRepo` для безпечного двотокенного збереження сесій (Access + Refresh токени) у базі даних PostgreSQL за стандартом UUID.
- Впроваджено централізований файл залежностей `src/services/dependencies.py` та асинхронну функцію `get_current_user_from_cookie` для розшифрування JWT-токенів із Cookies.
- Створено клас налаштувань `AuthSettings` та абстрактний клас `ModelConfig` для автоматичного зчитування параметрів JWT (`SECRET_KEY_JWT`, `ALGORITHM`) з файлу `.env`.
- Додано веб-маршрут для виходу із системи (`GET /auth/logout`), який анулює сесію, видаляє куки доступу та перенаправляє на головну сторінку.
- Додано захищений технічний ендпоінт (`POST /api/auth/logout`) для безпечного завершення сесії.
- Додано технічний ендпоінт (`POST /api/auth/refresh`) для автоматичного оновлення сесії без повторного введення пароля.
- Додано метод `verify_refresh_token` у сервіс автентифікації для криптографічної перевірки структури та терміну дії JWT токенів.
- Додано метод `get_token_record` у репозиторій користувачів для перевірки наявності активної сесії в таблиці `tokens` бази даних.
- Створено схему валідації `RefreshTokenRequestSchema` для безпечного приймання токенів оновлення.

### Changed
- Виконано архітектурне розділення роутів: створено структуру `src/web/` для рендерингу HTML та `src/routers/` для REST API.
- Очищено файл `main.py` від логіки рендерингу сторінок через перенесення головного екрана в модульний веб-роутер.
- Повністю оновлено базовий шаблон `base.html`: інтегровано верхню панель навігації (Navbar) з динамічним відображенням Email авторизованого користувача, кнопки "Вийти" або посилань для гостей ("Увійти" / "Реєстрація").
- Налаштовано автоматичне перенаправлення (Redirect) на головну сторінку `/` після успішного входу.
- Впроваджено механізм безпечної ротації токенів: при кожному оновленні сесії старий Refresh токен автоматично анулюється в базі даних та замінюється на новий.

## [0.3.0] - 2026-06-21

### Added
- Створено SQLAlchemy модель `Currency` (Валюта) з полями для трилітерного коду ISO, назви, прапорця базової валюти та курсу.
- Впроваджено тип `Numeric(10, 4)` (`Decimal` в Python) для збереження курсів валют для запобігання помилок округлення.
- Згенеровано та застосовано асинхронну міграцію Alembic для створення таблиці `currencies` у базі даних PostgreSQL.
- Створено шар збереження `CurrencyRepo` та бізнес-сервіс `CurrencyService` для повної ізоляції SQL-запитів від логіки додатку.
- Реалізовано повний технічний CRUD-цикл через REST API:
  - `POST /api/currencies/` — створення нової валюти.
  - `GET /api/currencies/{currency_id}` — пошук за кодом (регістронезалежний).
  - `PUT /api/currencies/{currency_id}` — оновлення назви та поточного курсу.
  - `PATCH /api/currencies/{currency_id}/set-main` — призначення валюти головною.
  - `DELETE /api/currencies/{currency_id}` — безпечне вилучення другорядної валюти.
- Створено Pydantic-схеми `CurrencyCreate`, `CurrencyUpdate` та `CurrencyResponse` з автоматичною валідацією коду у верхній регістр (`UPPERCASE`).
- Розроблено ізольовані фронтенд-сторінки на базі Jinja2:
  - `currencies.html` — загальний список валют у вигляді таблиці з кнопками дій.
  - `currency_add.html` — ультрамінімалістична форма додавання нової грошової одиниці.
  - `currency_edit.html` — захищена форма редагування чинних курсів з автоматичним блокуванням зміни бази відліку.
